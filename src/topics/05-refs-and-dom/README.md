# 05 Refs and DOM

Working with refs and imperative code in React.

## Contents

### Puzzle01: forwardRef + useImperativeHandle
- **Topic**: Exposing child methods to parents
- **Key Concept**: Control what's exposed, not just the DOM
- **Use case**: focus(), scrollIntoView(), animations

## Key Takeaways

1. Refs bypass React's declarative model
2. useImperativeHandle limits what's exposed
3. Use sparingly - prefer declarative patterns when possible
